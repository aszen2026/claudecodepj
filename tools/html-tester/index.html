How It Works

This tool uses an iframe element with the srcdoc attribute. When you edit the textarea, your HTML is injected directly into the iframe, creating a fully isolated rendering context. JavaScript executes inside the iframe's sandbox, isolated from this page. Console output is captured by overriding console methods inside the iframe and forwarding messages back via postMessage.

Frequently Asked Questions

Is my code uploaded to a server?
No. Everything runs entirely in your browser using an iframe sandbox. Your code never leaves your machine.
Can I use external libraries like jQuery or React?
Yes. Add a <script src="https://cdn..."></script> tag in the <head> or before your code. The iframe sandbox allows external network requests.
Does the console output show errors?
Yes. console.log, console.warn, and console.error from your code are captured and shown in the console panel below the preview.
Is there a file size limit?
Practically no — the browser handles hundreds of KB of HTML without issue. For very large projects, consider a full IDE.
Why does my code not run?
Check for syntax errors in the console output. Common issues: missing closing tags, unescaped quotes inside attributes, or <script> placement before the DOM element it references.
